# ТЗ / Промпт для реализации авторизации Firebase и переключения тем в Dancela

Данный документ содержит подробные технические требования и готовый промпт, который можно передать любому другому разработчику или ИИ-ассистенту для бесшовной интеграции авторизации и настроек темы.

## Готовый промпт для ИИ-ассистента / Разработчика

Копируйте текст ниже и вставьте его в начале следующей сессии разработки:

```markdown
Пожалуйста, реализуй в приложении Dancela полноценную систему авторизации через Firebase (Email, Google, Телефон) и систему выбора светлой/темной темы в настройках пользователя.

### ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

#### 1. Firebase Интеграция & Подключения
- Восстанови настоящую конфигурацию Firebase в `/src/lib/firebase.ts`, используя переменные окружения и `firebase-applet-config.json`.
- Подключи модули:
  - **Firebase Auth** для управления пользователями.
  - **Firestore / Realtime Database** для хранения данных пользователей и бронирований.
- Настрой экспорты необходимых функций: `auth`, `googleProvider`, `signInWithPopup`, `signInWithEmailAndPassword`, `createUserWithEmailAndPassword`, `signOut`, `onAuthStateChanged`.

#### 2. Полноценная авторизация (Email, Google, Телефон)
- Разработай заново UI-компонент `/src/components/Auth.tsx` с премиальным и стильным дизайном.
- В компоненте должны быть доступны 3 метода входа:
  1. **Google Sign-In**: Быстрый вход в один клик.
  2. **Email & Password**: Формы входа и регистрации (с валидацией длины пароля не менее 6 символов, проверкой занятости email).
  3. **Phone Auth (Вход по телефону)**: Интеграция Firebase Phone Auth с невидимой рекапчей (`RecaptchaVerifier`), отправкой SMS-кода через `signInWithPhoneNumber` и полем ввода OTP-кода подтверждения.

#### 3. Безопасная проверка прав администратора (Admin Role)
- После успешного входа пользователя система должна проверять, является ли он администратором.
- Пользователь получает роль `'admin'`, если его email или номер телефона (после очистки от лишних символов и приведения к международному формату) совпадает с одним из следующих списков:
  - **Административные Email**:
    - `Dancela2024@gmail.com`
    - `isabekdanial33@gmail.com`
  - **Административные Телефоны**:
    - `77064069886` (и любые эквиваленты: `+77064069886`, `87064069886`)
    - `87767722013` (и любые эквиваленты: `+77767722013`, `87767722013`)
    - `87014962932` (и любые эквиваленты: `+77014962932`, `87014962932`)
- **КРИТИЧЕСКОЕ ПРАВИЛО БЕЗОПАСНОСТИ**:
  - Нигде в интерфейсе (ни в плейсхолдерах, ни в хинтах, ни в подсказках "Доступ администратора:") НЕ должно упоминаться, какие именно почты и телефоны ведут к админ-панели. Проверка должна выполняться строго в кодовой части без раскрытия личных данных админов посторонним пользователям.

#### 4. Настройки пользователя и Переключение тем (Light & Dark Theme)
- В панели личного кабинета обычного пользователя добавь переключатель темы (Светлая / Темная).
- Сохраняй выбранную тему в `localStorage`, чтобы она сохранялась при перезагрузке страницы.
- Реализуй переключение через добавление/удаление класса `dark` у корневого элемента `document.documentElement` (или `document.body`).
- Убедись, что все разделы приложения корректно реагируют на изменение темы и стилизованы с использованием Tailwind CSS классов (например, `bg-white dark:bg-slate-950 text-slate-900 dark:text-white`).

#### 5. Интеграция в Навигацию
- Интегрируй кнопку "Войти" назад в компонент `Navbar.tsx` (как на десктопной, так и на мобильной версии), которая перенаправляет пользователя на страницу `/auth`.
- После успешного входа подставляй данные авторизованного пользователя в стейт `currentUser` приложения. При выходе сбрасывай стейт и вызывай `signOut(auth)`.
```

## Архитектурные подсказки для реализации

1. **Функция нормализации телефона для проверки админов**:
   ```typescript
   export const isUserAdmin = (email?: string, phone?: string): boolean => {
     const cleanEmail = email?.trim().toLowerCase();
     const cleanPhone = phone?.replace(/\D/g, '') || '';

     const adminEmails = ['dancela2024@gmail.com', 'isabekdanial33@gmail.com'];
     const adminPhones = ['77064069886', '87767722013', '87014962932'];

     if (cleanEmail && adminEmails.includes(cleanEmail)) {
       return true;
     }

     if (cleanPhone) {
       // Нормализуем номер к единому формату без +7, 7 или 8 в начале
       const rawDigits = cleanPhone.length > 10 ? cleanPhone.slice(-10) : cleanPhone;
       for (const adminPhone of adminPhones) {
         const adminRaw = adminPhone.slice(-10);
         if (rawDigits === adminRaw) {
           return true;
         }
       }
     }

     return false;
   };
   ```

2. **Внедрение Tailwind-переключения тем в React**:
   ```typescript
   // Вставьте этот хук или код инициализации в App.tsx
   useEffect(() => {
     const savedTheme = localStorage.getItem('dancela_theme') || 'dark'; // по умолчанию темная
     if (savedTheme === 'dark') {
       document.documentElement.classList.add('dark');
     } else {
       document.documentElement.classList.remove('dark');
     }
   }, []);
   ```
